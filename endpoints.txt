---
  Newsstand 1.1 — Mirror Server Documentation
  Python drop-in replacement for www.getnewsstand.com (Mac OS 9 / Classic)

  The original app (PowerPC RealBasic) made HTTP requests to www.getnewsstand.com.
  This Python server replicates all endpoints discovered through binary analysis
  and live traffic capture.

  SETUP: sudo python3 newsproxy.py
  Mac OS 9 HOSTS: add "10.0.3.x  www.getnewsstand.com" under Extensions > TCP/IP
  Port: 80 (HTTP — the app does not support HTTPS)

---
  XML FORMAT (CONFIRMED via live capture from getnewsstand.com)

  Article list (index.php / topic.php / feedlist.php):
    <?xml version="1.0" encoding="UTF-8"?>
    <League>
      <Player><name>Article title</name><position>http://www.getnewsstand.com/app/article.php?loc=US&amp;a=ARTICLE_URL</position></Player>
      ...
    </League>

  Individual article (article.php):
    <?xml version="1.0" encoding="UTF-8"?>
    <Player><name>Headline</name><position>Article body in plain text</position></Player>

  Content-Type: text/html; charset=UTF-8  (PHP default — NOT text/xml)

  How the app parses:
    - XQL "//Player" finds all <Player> elements
    - player.FirstChild.Value  -> text inside <name>  -> title in ListBox
    - player.LastChild.Value   -> text inside <position> -> URL or body
    - <position> must NEVER be empty: causes NilObjectException crash in the app

  Endpoints and Parameters

  Endpoint: /app/index.php
  Parameter: (none)
  Description: Sidebar menu with available sections (Today, Nation, World, etc.)
  ────────────────────────────────────────
  Endpoint: /app/index.php
  Parameter: loc
  Values: US, UK, JP, CA, DE, IT, FR, AU, TW, NL, BR, TR, BE, GR, IN, MX,
          DK, AR, CH, CL, AT, KR, IE, CO, PL, PT, PK, RU  (28 countries)
  Description: Filters news by country
  ────────────────────────────────────────
  Endpoint: /app/index.php
  Parameter: section
  Values: nation (National News), world (World News)
  Description: Filters by section; omitting this param returns "Today's News"
  ────────────────────────────────────────
  Endpoint: /app/topic.php
  Parameter: topic
  Values: Google News topic ID (CAAq... — base64-encoded protobuf)
  Description: Articles from a specific Google News topic
  ────────────────────────────────────────
  Endpoint: /app/feedlist.php
  Parameter: feed
  Values: RSS 2.0 or Atom feed URL, or site homepage (auto-discovery)
  Description: Custom RSS/Atom feed added by the user in Preferences.
               If the URL points to an HTML page, the server discovers the feed
               via <link rel="alternate" type="application/rss+xml"> automatically.
  ────────────────────────────────────────
  Endpoint: /app/article.php
  Parameter: loc  (country, e.g. US)
  Parameter: a    (article URL — can be a Google News URL or a direct URL)
  Description: Fetches and extracts the article text. For Google News URLs,
               decodes via the batchexecute API to obtain the real URL.
               Uses trafilatura to extract only the main body text.
               Fallback: uses the RSS feed body when the site blocks scraping.

  Technical notes

  Google News URL decoding (batchexecute):
    - Google News article URLs are redirects: news.google.com/rss/articles/<base64>
    - To obtain the real URL: fetch data-n-a-sg and data-n-a-ts from the article page,
      then POST to https://news.google.com/_/DotsSplashUi/data/batchexecute
      with f.req containing the Fbv4je parameters

  RSS/Atom feeds:
    - RSS 2.0: root <rss>, items under <channel>/<item>, link in <link> or <guid>
    - Atom:    root <feed> with namespace http://www.w3.org/2005/Atom,
               items in <entry>, link in the href attribute of <link>
    - content:encoded (namespace http://purl.org/rss/1.0/modules/content/) contains
      the full article body in many WordPress feeds
    - Malformed feeds (unescaped &, UTF-8 BOM): fixed before parsing;
      last resort: lxml with recover=True

  Sites that block scraping (Ars Technica AWS WAF, Cult of Mac 403):
    - The server caches the body of RSS feed items when loading the article list
    - When opening an article, it uses the feed cache instead of scraping
    - Works for any site that includes content in the feed itself

  Text for Mac OS 9:
    - Text fields use \r (CR) as line break — \n renders as a square glyph
    - Solution: flatten all whitespace to single spaces in the article body
    - Non-ASCII characters: transliterated via _ASCII_SUBS table (accents, cedilla,
      em dash, typographic quotes, ordinal symbols o. and a.)
